SLM Support API

A locally-running document intelligence API powered by Llama 3.2 1B and hierarchical RAG. Upload your documents, then query them from any application — no external API keys, no data leaving your server.

🦙 Llama 3.2 1B 🔍 Hierarchical RAG ⚡ GPU Accelerated 🔒 Fully Offline 📄 PDF · DOCX · TXT · MD
Configuration

Base URL

All API calls are made to your local server. By default, the Docker container exposes port 8000.

Base URL http://localhost:8000
💡 To expose this API to other machines on your network, use your server's local IP (e.g. http://192.168.1.x:8000) or configure a reverse proxy like Nginx in front of it.
Access

CORS & Authentication

The API is configured with open CORS — any origin can call it. There is no authentication by default since the server is typically only reachable on your local network or LAN.

⚠ If you expose this API to the public internet, you should add authentication middleware (e.g. an API key header) and restrict CORS origins in backend/main.py.
Tutorial

Quick Start

Get from zero to a working query in 4 steps.

1

Start the server

Make sure Docker is running with the NVIDIA GPU support, then start the container:

Shell
docker-compose up

Wait for the log line: SLM loaded successfully. — the model has been loaded onto the GPU and is ready.

2

Check the server is alive

Shell
curl http://localhost:8000/health
Response
{"status": "ok", "components": {"rag": "ready", "slm": "ready"}}
3

Upload a document

Shell
curl -X POST http://localhost:8000/v1/documents/upload \
  -F "files=@your_document.pdf"
Response
{"status": "success", "files_processed": ["your_document.pdf"], "chunks_indexed": 42}
4

Ask a question

Shell
curl -X POST http://localhost:8000/v1/query \
  -H "Content-Type: application/json" \
  -d '{"question": "What is the return policy?"}'
Response
{"answer": "Returns are accepted within 30 days...", "sources": ["your_document.pdf"]}
Endpoints

Endpoints

GET /health Server liveness check

Returns the current status of the API server and the SLM component. Use this to check if the model has finished loading before sending queries.

200 OK 
{
  "status": "ok",
  "components": {
    "rag": "ready",
    "slm": "ready"  // "loading" during startup
  }
}
POST /v1/documents/upload Upload & index documents

Accepts one or more files as multipart/form-data. The server extracts text, applies hierarchical chunking, embeds the chunks, and adds them to the FAISS vector store.

Request Body

Field Type Required Description
files File[] required One or more files. Supported: .pdf .docx .txt .md
200 OK 
{
  "status": "success",
  "files_processed": ["manual.pdf", "faq.txt"],
  "chunks_indexed": 87
}
POST /v1/query Query the knowledge base

The main inference endpoint. Searches the vector store for relevant context, then generates a grounded answer using the Llama 3.2 SLM.

Request Body

Field Type Required Description
question string required The natural language question to answer from the documents.

Response

Field Type Description
answer string The generated answer. If no relevant context found, returns a "don't know" response.
sources string[] Filenames of the documents that contributed context for this answer.
200 OK 
{
  "answer": "The warranty covers manufacturing defects for 2 years...",
  "sources": ["warranty_policy.pdf"]
}
POST /v1/reset Wipe the knowledge base

Deletes all indexed document vectors from the FAISS store and resets metadata. Use this to start fresh with a new set of documents.

200 OK 
{"status": "success", "message": "Index reset"}
Code Examples

Integration Examples

JavaScript / Fetch

JavaScript
// 1. Upload a document
async function uploadDocument(file) {
  const formData = new FormData();
  formData.append('files', file);

  const res = await fetch('http://localhost:8000/v1/documents/upload', {
    method: 'POST',
    body: formData
  });
  return res.json(); // { status, files_processed, chunks_indexed }
}

// 2. Ask a question
async function askQuestion(question) {
  const res = await fetch('http://localhost:8000/v1/query', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ question })
  });
  const data = await res.json();
  console.log(data.answer);   // the generated answer
  console.log(data.sources);  // ["filename.pdf"]
  return data;
}

Python / requests

Python
import requests

BASE_URL = "http://localhost:8000"

# 1. Upload a document
def upload_document(filepath: str):
    with open(filepath, "rb") as f:
        res = requests.post(
            f"{BASE_URL}/v1/documents/upload",
            files={"files": f}
        )
    return res.json()

# 2. Query the knowledge base
def ask(question: str) -> dict:
    res = requests.post(
        f"{BASE_URL}/v1/query",
        json={"question": question}
    )
    data = res.json()
    print(f"Answer: {data['answer']}")
    print(f"Sources: {data['sources']}")
    return data

# Example usage
upload_document("company_faq.pdf")
ask("What is the refund process?")

React — Custom Hook

JSX
import { useState } from 'react';

const SLM_BASE = 'http://localhost:8000';

export function useSLM() {
  const [loading, setLoading] = useState(false);
  const [answer, setAnswer] = useState(null);
  const [sources, setSources] = useState([]);

  const query = async (question) => {
    setLoading(true);
    try {
      const res = await fetch(`${SLM_BASE}/v1/query`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ question })
      });
      const data = await res.json();
      setAnswer(data.answer);
      setSources(data.sources);
    } finally { setLoading(false); }
  };

  return { query, answer, sources, loading };
}

// Usage in a component:
export function AIChat() {
  const { query, answer, sources, loading } = useSLM();
  return (
    <div>
      <button onClick={() => query("What is the pricing?")}>Ask</button>
      {loading && <p>Thinking...</p>}
      {answer && <p>{answer}</p>}
      {sources.map(s => <small key={s}>Source: {s}</small>)}
    </div>
  );
}

Reference

Error Codes

Status Meaning Common Cause
200 Success Request completed normally.
422 Validation Error Missing required field in request body (e.g., no question field).
500 Server Error Model not loaded yet, or internal exception. Check Docker logs.
ERR_CONNECTION_REFUSED Network Error Docker container not running, or wrong port.

Limits & Tips

  • 📄 Supported file types: .pdf, .docx, .txt, .md
  • ⚡ Response time: 1–5 seconds on GPU. First query after startup may be slower as the GPU warms up.
  • 💾 Model persists: The downloaded 800 MB GGUF model is cached in ./models/slm/ and survives rebuilds.
  • 📚 Index persists: Uploaded documents survive container restarts — they're stored in ./data/vector_store/.
  • 🔄 Multi-file upload: You can upload multiple files in a single POST request by appending multiple files fields.
  • 🧹 Changing documents: Call POST /v1/reset before uploading a completely new set of unrelated documents to avoid mixed context.
  • 🔍 Answer grounding: The SLM will say "I don't know based on the provided documents" if the answer isn't in any uploaded file — it won't hallucinate.
